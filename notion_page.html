Docker

가상머신은 os위에 os를 띄우는 방식, 도커는 자신만의 컨테이너 가상화기술을 이용해서 os위에 정리된 프로세스를 띄워줌

위에사진에서 1번부터 21번라인까지는 이미지를 빌드하기위한 커맨드들이다.

22번부터는 이미지가 실행되서 컨테이너가 됬을때 실행되는 커맨드

맨뒤에 . 은 반드시 붙혀줘야 함

-p 는 포트를 포트포워딩을 해주는것.

docker push - 나의도커허브에 이미지파일을 올리는것

그렇게해서 올린 이미지파일은 누구나 받아갈수있음

수정사항시생기면 태크버전을 0.1.1 로 변경 후 다시 도커허브에올리면된다.

풀을 받은다음에 런

// 이미지 하나의 여러개의 컨테이너를 띄울수있음

하나의 이미지파일로 여러개의 컨테이너를 띄울수있음

이미지 하나로 요청이많아지면 같은 컨테이너를 여러개로 만들어서 분산시킬수있음.

로컬에서 이미지를 만들고 개인도커허브에 올리고

ec2서버에서 도커허브에서 내려받아서 런만시키면 됨

도커의장점

  1. MSA에 적합한 방식
  1. 배포하면서 발생할수있는 많은 버그와 오류들이 많이 줄어듬
  1. 환경변화에 적응이빠름

도커파일은 도커만의 문법으로 작성

그 이미지를 가지고 띄운것이 컨테이너

이미지 = 구글크롬을 클릭하기 전

컨테이너 = 구글크롬을 클릭하는 동시에 실행되는 것

도커의 단점

  1. 도커는 호스트OS에 굉장히 의존성을 많이 가지고있음
  1. 호스트 OS가 리눅스컨테이너 여야만 도커컨테이너를 띄울수 있음

맥OS에서 할때 가상머신으로 먼저

CI/CD 환경 (Github) 에 모든파이프라인이 도커를통해서 배포됨.
